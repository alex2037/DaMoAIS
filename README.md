# Развертывание и сопровождение систем ИИ.

> **Статус:** в разработке  
> **Версия:** 0.1.0  
> **Последнее обновление:** 2026-07-13

Краткое описание дисциплины: В рамках подготовки кадров по направлению ИИ имеем сильный перекос в "математиков" (людей-мечтателей, которые занимаются фронтирными задачами), "вайбкодеров" (пользователи ИИ инструментов, думая, что они умеют программировать). Но при желании подготовки инженеров (а именно инженерные задачи бизнеса приносят деньги) необходимо выдавать реальные задачи применения ИИ. Причем немаловажным местом применения ИИ является совместное использование серверных (БЯМ требуют много ресурсов) локальных (личные данные компании) моделей на группу лиц для различных целей (программирование, RAG системы, ведение проектов и другое). Хочется показать студентам как с этим жить. Ключевой акцент — переход от «запустил локально — работает» к «запустил в проде — выдерживает нагрузку группы из 30 человек». 

## Цель и задачи дисциплины

- **Цель:** сформировать у обучающихся …
- **Задачи:**
  - изучить …
  - освоить …
  - научиться …

## Структура репозитория

```
DaMoAIS/
├── README.md                    # Общее описание
├── docs/                        # методические и вспомогательные материалы
│   ├── glossary.md              # термины и определения
│   ├── bibliography.md          # основная и дополнительная литература
│   └── ...
├── syllabus/                    # рабочая программа дисциплины (РПД)
│   └── README.md
├── labs/                        # лабораторные работы
│   ├── lab_01/
│   ├── lab_02/
│   └── ...
├── assessment/                  # фонд оценочных средств (ФОС)
│   └── README.md
```

## Состав учебно-методического комплекса

| Раздел | Статус | Описание |
|--------|--------|----------|
| Рабочая программа дисциплины (РПД) | 🚧 в планах | Объём, цели, результаты освоения, тематический план, содержание разделов |
| Методические указания / документация | 🚧 в планах | Теоретический минимум, примеры, чек-листы |
| Лабораторные работы | 🚧 в планах | Описания лаб, задания, критерии приёмки, шаблоны отчётов |
| Фонд оценочных средств (ФОС) | 🚧 в планах | Вопросы к зачёту/экзамену, тесты, задания, шкалы оценивания |

## План работы

- [ ] Определить название, цель и профиль обучающихся
- [ ] Утвердить структуру РПД и заполнить разделы
- [ ] Подготовить теоретическую документацию по каждой теме
- [ ] Разработать лабораторные работы с заданиями и критериями
- [ ] Сформировать ФОС: вопросы, тесты, задания, критерии оценивания
- [ ] Провести вычитку и привести оформление к единому стилю

## Соглашения по оформлению

- Файлы `.md` оформляются в формате Markdown.
- Каждый раздел хранится в отдельной директории.
- Изображения и вложения размещаются в подпапке `assets/` рядом с документом, где они используются.
- Статусы:
  - 🚧 в планах
  - 📝 в работе
  - ✅ готово
  - 🔄 на проверке

## Как работать над материалами

1. Открывай `README.md` — здесь актуальный статус и план.
2. Переходи в нужный раздел (`syllabus/`, `labs/`, `assessment/`, `docs/`).
3. Редактируй `.md`-файлы, добавляй новые файлы по мере необходимости.
4. Обновляй статусы в таблице выше и отмечай выполненные пункты плана.
