# Методические указания преподавателям по использованию ресурсов

Указания описывают, как применять материалы из каталога [`resources/`](../../resources/README.md) при разработке и проведении дисциплины: ручное использование, автоматизированную обработку (с примерами кода) и генерацию учебных материалов с помощью БЯМ, а также как поддерживать ресурсы в актуальном состоянии.

## 1. Что лежит в `resources/`

Ресурсы сгруппированы по типам; каждый описывается по [единой карточке ресурса](../../resources/README.md#единая-карточка-ресурса) (название, тип, аннотация, связанные КИМ, доступ, лицензия, дата проверки).

| Тип | Раздел | Для чего использовать при разработке |
|---|---|---|
| Учебники | [textbooks](../../resources/textbooks/README.md) | опорные материалы лекций и теоретической части ЛР |
| Научные статьи | [papers](../../resources/papers/README.md) | актуальное состояние области, задания продвинутого уровня (П) |
| Банки тестов | [test-banks](../../resources/test-banks/README.md) | экспресс-тесты (промежуточное тестирование) |
| Банки задач | [problem-banks](../../resources/problem-banks/README.md) | постановки для ЛР и практических заданий |
| Датасеты | [datasets](../../resources/datasets/README.md) | данные для RAG-контура и нагрузочных сценариев |
| Бенчмарки | [benchmarks](../../resources/benchmarks/README.md) | эталоны для сравнения конфигураций и A/B |
| Python-библиотеки | [software/python-libs](../../resources/software/python-libs/README.md) | инструменты реализации и автопроверки |
| LLM и промпты | [llm-prompts](../../resources/llm-prompts/README.md) | генерация, анализ и оценивание с применением БЯМ |
| Прочее | [other](../../resources/other/README.md) | справочники, сервисы, вспомогательные материалы |

## 2. Ручное использование

- **Привязка к КИМ.** Подбирая ресурс, заполняйте поле «Связанные КИМ» относительной ссылкой на модуль (`M1`…`M8`) — это поддерживает триангуляцию (каждый индикатор проверяется ≥ 2 средствами).
- **Уровни КРМ.** Учебники и типовые постановки — для базового/среднего уровня (Б/С); статьи и бенчмарки — для продвинутого (П), где требуется сравнение альтернатив и обоснование.
- **Лицензионная чистота.** Перед включением материала в задание проверяйте поле «Лицензия»: в открытый репозиторий (CC BY 4.0) нельзя вносить материалы с несовместимыми условиями; на такие ресурсы давайте ссылку, а не копию.
- **Проектный сквозной сценарий.** Для ЛР4 (RAG) и ЛР7 (нагрузка) подбирайте датасеты и бенчмарки, соответствующие профилю «30 пользователей», чтобы измерения были сопоставимы между работами.

## 3. Автоматизированное использование

### 3.1 Проверка ссылок и полноты карточек

Скрипт-обходчик находит незаполненные маркеры и битые внешние ссылки во всех карточках ресурсов. Пример на Bash (требует `grep` и `curl`):

```bash
#!/usr/bin/env bash
# Проверка карточек ресурсов: маркеры [ЗАПОЛНИТЬ] и доступность внешних ссылок.
set -uo pipefail
ROOT="resources"

echo "== Незаполненные маркеры =="
grep -rn "\[ЗАПОЛНИТЬ\]" "$ROOT" || echo "маркеров не найдено"

echo "== Проверка внешних ссылок =="
grep -rhoE "https?://[^) ]+" "$ROOT" | sort -u | while read -r url; do
  code=$(curl -s -o /dev/null -w "%{http_code}" -m 10 -L "$url")
  printf "%s\t%s\n" "$code" "$url"
done
```

### 3.2 Инвентаризация ресурсов из карточек

Пример на Python: собирает таблицы карточек в единый реестр и подсвечивает записи со старой датой проверки.

```python
#!/usr/bin/env python3
"""Сводный реестр ресурсов с проверкой давности поля «Дата проверки»."""
import re, sys, datetime, pathlib

ROOT = pathlib.Path("resources")
STALE_DAYS = 180
today = datetime.date.today()
row_re = re.compile(r"^\|(.+)\|\s*$")
date_re = re.compile(r"\d{4}-\d{2}-\d{2}")

for md in ROOT.rglob("README.md"):
    for line in md.read_text(encoding="utf-8").splitlines():
        m = row_re.match(line)
        if not m or "ЗАПОЛНИТЬ" in line or "Название" in line or "---" in line:
            continue
        cells = [c.strip() for c in m.group(1).split("|")]
        d = date_re.search(cells[-1]) if cells else None
        if d:
            age = (today - datetime.date.fromisoformat(d.group())).days
            flag = "STALE" if age > STALE_DAYS else "ok"
            print(f"{flag}\t{age:>4}d\t{md.parent.name}\t{cells[0]}")
```

Такие скрипты удобно запускать в CI (шаг проверки перед публикацией) вместе с [чек-листом качества](../../docs/quality-checklist.md).

### 3.3 Промпты для БЯМ

Ресурсы из [`llm-prompts`](../../resources/llm-prompts/README.md) применяются для генерации черновиков заданий, тестовых вопросов и разборов. Правило: **БЯМ готовит черновик, преподаватель верифицирует и адаптирует**. Фиксируйте использованные промпты в карточке ресурса.

Пример промпта для генерации экспресс-теста по модулю:

```text
Роль: методист по инженерии эксплуатации ИИ-систем.
Контекст: модуль «<название>», целевой индикатор КРМ <код>, уровень <Б/С/П>.
Задача: составь 5 вопросов с одним верным ответом для экспресс-теста в начале лекции.
Требования: проверяй понимание, а не запоминание; варианты-дистракторы правдоподобны;
для каждого вопроса дай верный ответ и одно предложение с обоснованием.
Формат: Markdown-таблица (вопрос | варианты | верный ответ | обоснование).
```

Пример промпта для черновика постановки ЛР:

```text
Роль: инженер по развёртыванию ИИ-сервисов.
На основе темы «<тема ЛР>» и сквозного проекта (shared ИИ-сервис на 30 пользователей)
предложи постановку по уровням КРМ (Б → С → П, каждый включает предыдущий),
список проверяемых артефактов и черновик 10-балльной рубрики.
Не выдумывай метрики: обозначь места, где преподаватель подставит целевые SLO.
```

После генерации обязательно: сверьте факты и версии инструментов, уберите «галлюцинации», приведите формулировки к [шаблону КИМ](../../docs/kim-template.md).

## 4. Обновление и проверка ресурсов

- **Дата проверки.** В каждой карточке поле «Дата проверки» (ГГГГ-ММ-ДД) обновляется при любой ревизии ресурса; записи старше ~6 месяцев считаются устаревшими и требуют перепроверки (см. скрипт 3.2).
- **Актуальность ссылок.** Перед началом семестра прогоняйте проверку ссылок (скрипт 3.1); битые внешние ссылки заменяйте или архивируйте (например, ссылкой на веб-архив), не удаляя запись бесследно.
- **Версии инструментов.** Для быстро меняющихся инструментов (vLLM, Triton, k6, ArgoCD и т. п.) фиксируйте проверенную версию в аннотации; при обновлении курса проверяйте, что задания и промпты не опираются на устаревший API.
- **Лицензии.** При обновлении перепроверяйте условия использования — они меняются; несовместимые с CC BY 4.0 материалы держите как внешние ссылки.
- **Синхронизация с КИМ.** После изменения ресурса проверьте обратные ссылки «Связанные КИМ» и соответствие модели измерения; используйте [чек-лист качества](../../docs/quality-checklist.md) как финальный контроль перед публикацией.
